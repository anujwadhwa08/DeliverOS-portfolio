Problems Solved
The Challenge What Was Built The Outcome
People think in cravings — "something light", "spicy tonight", "quick and under $10" — not in category filters and search keywords. AI converts natural-language queries into meaning-based searches across live restaurant menus. Customers order by craving, not by knowing the right category to tap.
Generative AI can produce plausible-sounding details that don't exist: restaurant names, prices, or menu items invented on the fly. Every AI response is verified by a second AI before reaching the user, blocking any invented content. Users can trust every restaurant, price, and time shown — it was verified before they saw it.
Coordinating a live order across a restaurant, a driver, and a customer involves real-time decisions across three separate parties. Three AI assistants share live order state and are each notified automatically when it's their turn to act. End-to-end coordination runs autonomously — from placement to delivery confirmation.
Every delivery has failure modes — driver no-shows, restaurant closures, disputed handovers — each requiring a defined recovery path. Every failure scenario is a defined state with automatic recovery: timeout triggers reassignment, cancel triggers refund. No order goes silent. Every customer gets a response. Every penny is accounted for.
Agentic commerce requires an API contract external agents can call — but most platforms assume a human with a browser and an existing account. And blindly linking by email would let any agent impersonate any user. MCP endpoint with two paths: new customers auto-onboard in one call (with explicit consent gate); existing customers verify via OTP email → receive a 15-min signed token → agent retries order. Any AI agent can place a real, tracked order. Existing accounts are protected by OTP verification — a malicious agent cannot impersonate a customer just by knowing their email.
Getting a restaurant live requires hours of manual data entry — re-typing every item, price, and category from a printed menu or website. Three menu import methods: paste a website URL (AI scrapes the page), photograph a physical menu (AI reads the image), or upload an Excel template. All three produce the same structured menu. A restaurant with a 100-item menu can go live in under five minutes — with zero manual data entry.

Designing with AI as a product capability, not a feature

Every AI decision in DeliverOS solved a user problem. Semantic search exists because customers don't browse like a database. The safety check exists because users can't tell when an AI is making something up. Three separate agents exist because a single generic AI gives worse answers to everyone. The MCP endpoint exists because agentic commerce is real — a customer's AI assistant should be able to order dinner the same way a human taps a button.

The hardest product work was designing the failure paths. When a restaurant doesn't respond, when a driver goes offline mid-delivery, when a customer disputes — every one is a defined state with a documented resolution. That's the difference between a demo and a platform.
